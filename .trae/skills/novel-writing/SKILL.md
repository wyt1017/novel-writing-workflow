# 小说创作工作流 Skill

## 概述

本 Skill 是小说创作全链路的入口，基于 `novel-writing-workflow-v2.md` 工作流规范。

**工作流程：** 选平台 → 选类型 → 选题材 → 定篇幅 → 设计爽点 → 故事结构 → 角色设计 → 世界观 → 大纲 → 写作 → 修订 → 发布

**24 个节点概览：**

| 阶段 | 节点 | 产出物 |
|------|------|--------|
| 一：平台与题材 | W01-W04 | 平台选择、类型确认、题材选择、方向确认书 |
| 二：故事框架 | W05-W06 | 爽点设计、故事结构 |
| 三：角色与世界观 | W07-W12 | 主角档案、配角反派、关系图、世界观、力量体系、地图势力 |
| 四：大纲与节奏 | W13-W16 | 开篇设计、悬念铺设、大纲规划、节奏规划 |
| 五：写作与迭代 | W17-W22 | 第1章、2-3章、读者测试、修订、持续写作、卷检 |
| 六：修订与发布 | W23-W24 | 全书终检、发布准备 |

---

## 触发条件

当用户提出以下需求时自动激活：
- "我要写小说"
- "帮我设计一个故事"
- "帮我规划一部小说"
- "开始创作"
- 任何涉及小说创作全流程的请求

---

## 强制执行规则

以下规则必须严格遵守，不得跳过任何步骤：

1. **严格按顺序执行：** W01 → W02 → W03 → W04 → ... → W24
2. **禁止跳过 W02（选作品类型）：** 必须先让用户选择作品类型，然后才能进入 W03
3. **W03 依赖 W02 的输出：** 题材推荐必须基于"平台 + 作品类型"
4. **每个节点必须用户确认后才能进入下一步：** 禁止合并多个节点
5. **禁止提前展示后续节点内容**

---

## 设计确认与保存流程（每个节点通用）

```
【设计完成】→ 【展示设计方案】→ 【询问用户是否调整】
                                    ↓
                      ┌─────────────┴─────────────┐
                      ↓                           ↓
                 【用户要求调整】            【用户确认不调整】
                      ↓                           ↓
                修改设计方案                保存设计到文件
                重新展示确认                  生成.md文档
```

## 工作流文档

完整的工作流规范文件：`novel-writing-workflow-v2.md`

该文档包含每个节点的详细执行步骤、验收标准（打分制）、硬性条件、产出物模板以及四级检查体系（章检/5章检/卷检/全书终检）。

---

## 启动方式

1. 用户说"开始"或直接描述创作需求
2. 从 W01 开始，发送问卷收集基本信息
3. 按顺序推进，每节点确认后保存文件再进下一步

---

## 注意事项

1. **保持原作意图**：在设计阶段充分尊重用户的选择
2. **尊重体裁风格**：不同平台/类型有不同的市场偏好
3. **适度原则**：不要过度设计导致迟迟无法动笔
4. **保存第一**：每个节点确认后必须保存为 .md 文件
5. **读取设定文件**：写作前必须先读取之前保存的所有设定文件
