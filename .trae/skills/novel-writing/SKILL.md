# 小说创作工作流 Skill（流畅模式 v2.1）

## 概述

本 Skill 是小说创作全链路的入口，基于 `novel-writing-workflow-v2.md` 工作流规范。

**24 节点概览：**

| 阶段 | 节点 | 产出物 |
|------|------|--------|
| 一：平台与题材 | W01-W04 | 平台选择、类型确认、题材选择、方向确认书 |
| 二：故事框架 | W05-W06 | 爽点设计、故事结构 |
| 三：角色与世界观 | W07-W12 | 主角档案、配角反派、关系图、世界观、力量体系、地图势力 |
| 四：大纲与节奏 | W13-W16 | 开篇设计、悬念铺设、大纲规划、节奏规划 |
| 五：写作与迭代 | W17-W22 | 第1章、2-3章、读者测试、修订、持续写作、卷检 |
| 六：修订与发布 | W23-W24 | 全书终检、发布准备 |

---

## ⚠️ 流畅模式 v2.1（核心变更）

原 v2.0「每个节点强确认」导致 24 次打断、体验卡顿。v2.1 改为**四阶段批处理 + 关键节点强确认**：

| 阶段 | 覆盖节点 | 交互方式 |
|------|----------|----------|
| A 立项 | W01-W04 | 一次问卷 → 一次确认 |
| B 设定 | W05-W12 | 分两批展示（B1:W05-06 / B2:W07-12）→ 两次确认 |
| C 大纲 | W13-W16 | 一次展示 → 一次确认 → 生成速查卡 |
| D 写作 | W17-W24 | 连续写作，章检自动跑，**零打断** |

**硬性约束（保留 v2.0 的严谨）：**
1. 平台 / 类型 / 题材 / 篇幅 四类信息必须全部收集，不得用默认值硬猜
2. 每个阶段产物必须先保存 .md 再进入下一阶段
3. 写作阶段每章必须过 novel-de-ai 章检（W21）
4. 不得提前展示下一阶段内容（阶段内可批量展示）

---

## 进度追踪机制（断点续跑）

- **每次完成一个阶段**，更新 `输出/进度追踪.md`（不存在则创建）
- 文件含：当前阶段、已完成节点、关键决策摘要、产物清单、下一步指令
- **Skill 启动第一步**：先读 `输出/进度追踪.md`。若存在，从断点继续；若不存在，从 W01 开始
- 用户换会话 / 中断后重开，无需重新走完设计

---

## 设定速查卡机制（写作加速）

- **W16 完成后自动生成 `输出/设定速查卡.md`**：单文件浓缩全部设定（主角一句话、爽点模式、五幕节点、力量当前值、前30章钩子、配角一句话、世界观、禁忌）
- **写作阶段（W17+）只读两份**：`输出/设定速查卡.md` + 当前正在写的 `输出/第N章.md`
- 不再逐篇读取 12+ 个分散设定文件，避免上下文过载、响应变慢
- 若用户中途调整设定：改原文件 + 同步更新速查卡

---

## 四阶段执行指令（AI 直接照做，无需翻 2000 行大文档）

### 阶段A 立项（W01-W04）
1. 发一份问卷：写作经验 / 偏好类型 / 日更能力 / 希望风格 / 平台倾向 / 篇幅预期
2. 基于回答推荐 平台→类型→题材→篇幅，一次性生成四个文件：
   - `输出/平台选择.md`、`输出/作品类型确认.md`、`输出/题材选择.md`、`输出/方向确认书.md`
3. 展示方案 → 等用户一次确认 / 微调 → 更新进度追踪 → 进入 B

### 阶段B 设定（W05-W12）
1. **B1（W05-W06）**：生成 `输出/爽点设计.md`（爽点模式+密度+反套路）、`输出/故事结构.md`（五幕式+关键节点）→ 展示 → 确认
2. **B2（W07-W12）**：生成 `输出/主角档案.md`、`输出/配角与反派档案.md`、`输出/人物关系图.md`、`输出/世界观设定.md`、`输出/力量体系.md`、`输出/地图与势力.md` → 展示 → 确认
3. 某节点需调整时仅改该节点后重展；全部确认后更新进度追踪 → 进入 C

### 阶段C 大纲（W13-W16）
1. 生成 `输出/开篇设计.md`（黄金三章规划）、`输出/悬念铺设.md`、`输出/大纲规划.md`（前30章逐章）、`输出/节奏规划.md`
2. 展示 → 确认 → **生成 `输出/设定速查卡.md`** → 更新进度追踪 → 进入 D

### 阶段D 写作（W17-W24）
1. 写第 N 章：读 `设定速查卡.md` → 写 `输出/第N章.md`（2000-3000字）
2. 每章写完自动章检：调用 `novel-de-ai` 检测并改写（不打断用户）
3. 每 3 章做一次读者视角测试（W19）→ 列 P0/P1/P2 → 自动修订（W20）
4. 持续写作按 W21 流程；每卷结束做卷检（W22）
5. 完本后全书终检（W23）+ 发布准备（W24）
6. 每完成若干章更新进度追踪

## 注意事项

1. **进度优先**：启动先读 `输出/进度追踪.md`，断点续跑
2. **速查卡优先**：写作只读 `设定速查卡.md` + 当前章节，不逐篇读全部设定
3. **保存第一**：每阶段产物必须落盘 .md
4. **风格尊重**：不同平台/类型有不同市场偏好
5. **不偏离**：写作前读速查卡，确保正文不偏离设定
