## ⚠️ 强制执行规则（必读）

**以下规则必须严格遵守，不得跳过任何步骤：**

1. **必须严格按顺序执行：** W01 → W02 → W03 → W04 → ... → W24
2. **禁止跳过W02（选作品类型）：** 必须先让用户选择作品类型（玄幻/都市/科幻/历史/游戏/悬疑/言情/纯爱/幻想），然后才能进入W03（选题材）
3. **W03依赖W02的输出：** 题材推荐必须基于"平台 + 作品类型"进行，没有作品类型就无法推荐合适的题材
4. **每个节点必须用户确认后才能进入下一步：** 禁止一步到位，禁止合并多个节点
5. **禁止提前展示后续节点内容：** 在执行W01时，只能讨论平台选择，不能提前讨论类型、题材等内容

**违反规则的后果：**
- 跳过W02会导致题材推荐不准确（因为没有根据作品类型来推荐）
- 用户可能得不到适合自己的题材建议
- 最终作品可能与用户兴趣不符

**正确流程示例：**
```
✅ 正确：W01选平台 → W02选类型（用户选择） → W03根据类型选题材
❌ 错误：W01选平台 → 直接推荐题材（跳过W02）
```

## 📋 设计确认与保存流程（每个节点通用）

**每个设计节点执行完毕后，必须按照以下流程操作：**

```
【设计完成】→ 【展示设计方案】→ 【询问用户是否调整】
                                      ↓
                        ┌─────────────┴─────────────┐
                        ↓                           ↓
                   【用户要求调整】            【用户确认不调整】
                        ↓                           ↓
                  修改设计方案                保存设计到文件
                  重新展示确认                  生成.md文档
                        ↓                           ↓
                        └─────────────┬─────────────┘
                                      ↓
                              【停止并等待确认】
```

**保存文件清单（后续写正文时必须读取）：**
- `平台选择.md` - 发布平台信息
- `作品类型确认.md` - 作品类型选择
- `题材选择.md` - 题材、标签、核心卖点
- `方向确认书.md` - 篇幅、故事设定、冲突
- `爽点设计.md` - 爽点模式和密度
- `故事结构.md` - 三幕/五幕式结构
- `主角档案.md` - 主角性格、动机、弧光
- `配角与反派档案.md` - 配角和反派设定
- `人物关系图.md` - 人物关系网络
- `世界观设定.md` - 时代/地理/社会/文化
- `力量体系.md` - 境界划分和战力
- `地图与势力.md` - 地图扩展和势力关系

**⚠️ 重要：写正文前必须先读取以上所有已保存的文件，确保不偏离设定！**

**📌 统一流程说明：所有节点（W01-W24）都遵循相同的"确认-调整-保存"流程**

每个设计节点完成后都会：
1. 展示设计方案
2. 询问用户是否需要调整
3. 用户确认后保存到对应的.md文件
4. 停止并等待用户确认后才进入下一节点

**所有已保存的设计文件，在W17开始写正文前都必须读取，确保正文不偏离设定！**

## 篇幅分支说明

**从W03选择篇幅后，以下节点会根据篇幅类型有不同的标准：**

| 节点 | 短篇/中篇标准 | 长篇/超长篇标准 |
|------|--------------|----------------|
| W05 结构设计 | 三幕式，单线/双线，3-5个转折点 | 五幕式或多卷，多线交织，≥5个转折点 |
| W10 力量体系 | 简化版，3-5个境界即可 | 完整版，≥5个大境界 |
| W11 地图势力 | 1-2个主要地图，2-3个势力 | ≥3个地图扩展，3-5个主要势力 |
| W14 章节大纲 | 前10章详细大纲 | 前30章详细大纲 |
| W15 节奏规划 | 每5章一个小高潮 | 每2-3章一个小高潮 |
| W20 持续写作 | 每5章阶段验收 | 每10章阶段验收 |
| W21 卷间修订 | 每卷修订（2-3万字） | 每卷修订（10-15万字） |
| W22 全书终检 | 全文通读+逻辑检查 | 九项逐章检查 |

**⚠️ 四级检查体系对所有篇幅通用：**
1. **章检** — 每章写完必检（10项标准，≥80分通过）
2. **5章检** — 每5章阶段检（5项标准，≥40/50分通过）
3. **卷检** — 每卷完结检（6项标准，≥50/60分通过）
4. **全书终检** — W22节点（九项检查，≥85/100分通过）

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

**向用户发送以下完整问卷（一次性提问）：**

```

请填写以下信息，我会根据你的情况推荐合适的平台：

1. 写作经验：
   □ 新手（没写过或只写过短篇）
   □ 有经验（写过完整故事≥5万字）

2. 偏好类型：
   □ 男频
   □ 女频
   □ 无偏好

3. 日更能力：
   □ 500-1000字（新手/兼职）
   □ 1500-3000字（进阶）
   □ 3000字以上（专业/全职）

4. 希望写的作品风格（可选）：
   □ 快节奏爽文
   □ 传统玄幻/仙侠
   □ 都市异能
   □ 其他：_______

收到你的回复后，我会：
- 推荐1-2个最适合的平台
- 说明推荐理由
- 让你确认最终选择
```

**步骤 1.2：根据用户回复推荐平台**
- 根据用户填写的问卷，推荐1-2个平台
- 说明推荐理由和平台特征
- 让用户从推荐中选择1个平台

| 平台 | 适合人群 | 核心特征 | 字数要求 | 签约难度 | 收入模式 |
|------|----------|----------|----------|----------|----------|
| 番茄小说 | 新手首选 | 免费阅读+算法推荐，流量大 | 100万+ | 易 | 广告分成 |
| 七猫小说 | 新手首选 | 免费阅读，门槛低 | 100万+ | 易 | 广告分成 |
| 起点中文网 | 有经验者 | 付费阅读，读者质量高 | 200万+ | 难 | 订阅+打赏 |
| 晋江文学城 | 女频作者 | 言情/纯爱为主 | 50万+ | 中 | 订阅+实体 |
| 飞卢小说网 | 快手老手 | 日更要6000+，脑洞文 | 100万+ | 极易 | 订阅分成 |
| 纵横中文网 | 传统作者 | 传统玄幻/仙侠 | 150万+ | 中 | 订阅+全勤 |
| 塔读文学 | 脑洞作者 | 轻小说/系统流 | 80万+ | 易 | 订阅分成 |

- 新手+男频 → 番茄/七猫
- 新手+女频 → 晋江/番茄
- 有经验+男频 → 起点/飞卢（看更新能力）

**步骤 1.2：确认平台选择**

**常见问题：**

  A：新手去起点/晋江容易被劝退，建议先在番茄/七猫练手，等有作品经验再转
- Q：飞卢日更6000我能做到吗？

### 【W02】选作品类型

### 【W05】核心爽点设计

**步骤 4.1：爽点模式选择**
- 从以下6大类中选择2-3个作为核心爽点模式：

**① 实力碾压类**
- 扮猪吃虎：隐藏实力→被轻视→展露实力→碾压反转
- 越级反杀：以弱对强→绝境翻盘→超越极限
- 科技碾压：高维科技vs低维文明→降维打击
- 信息差碾压：掌握关键信息→利用信息差取胜

**② 身份反转类**
- 身份觉醒：隐藏身份揭露→众人震惊→态度反转
- 废柴逆袭：天赋被夺/废物→奇遇金手指→逐步崛起
- 退婚流：被退婚受辱→崛起→对方后悔
- 兵王回归：退役隐居→被迫出手→王者归来

**③ 智慧谋略类**
- 智商碾压：布局算计→对手入彀→真相揭露
- 幕后黑手：暗中操控→局势按计划发展→身份揭露
- 迪化流/误解向：行为被过度解读→他人脑补→效果超预期

**④ 稳健成长类**
- 稳健流/苟道：低调发育→避免风险→厚积薄发
- 签到/躺赢：签到获奖励→轻松变强→不劳而获
- 经营致富：从零开始→商业经营→财富积累

**⑤ 情感关系类**
- 后宫/修罗场：多角色围绕→争风吃醋→主角周旋
- 万人迷/团宠：众人喜爱→被宠溺→集万千宠爱
- 虐渣复仇：被伤害→积蓄力量→复仇打脸

**⑥ 特殊机制类**
- 爆兵平推：数量碾压→以量取胜
- 第四天灾/玩家流：玩家涌入→混乱创造→不可预测
- 直播崇拜：直播展示→观众震惊→打赏崇拜
- 气运流：气运加身→机缘不断→天命所归

**步骤 4.2：爽点结构设计**
- 为主选爽点设计三段式结构：
  - 铺垫期（30%）：建立对手的嚣张/主角的困境
  - 爆发期（50%）：反转/打脸/实力展现
  - 余韵期（20%）：众人反应/后续影响/主角淡然
- 每个爽点必须设计具体场景

**步骤 4.3：爽点密度规划**
- 根据目标平台设定爽点密度：

| 平台 | 小爽点间隔 | 大爽点间隔 |

8. **失败成长** — 主角经常失败，从失败中学习
9. **反英雄** — 主角有道德缺陷，不是传统好人
10. **开放式结局** — 不给出明确答案，让读者思考

- 真实性：10分
- 精炼度：10分

#### 第一级：每章写完必检（章检）

- 必须≥80分才能写下一章
- 不通过必须当场修改

#### 第二级：每5章阶段检（5章检）
- 每完成5章，进行一次阶段检查
- 检查范围：最近5章
- 检查内容：
  - 5章内设定一致性（角色/力量体系/地图）
  - 5章内节奏是否合理
  - 爽点密度是否符合规划
  - 伏笔是否有遗漏或需要回收
  - 5章之间的连贯性
- 检查评分（5项每项0-10分，总分≥40/50分通过）
- 不通过必须修改后再继续

#### 第三级：每卷完结检（卷检）
- 每完成1卷（通常30-50章），进行一次卷检
- 检查范围：整卷
- 检查内容：

  - 整卷伏笔回收情况
  - 整卷悬念解答情况
  - 整卷高潮是否到位
  - 是否需要调整后续大纲
- 检查评分（6项每项0-10分，总分≥50/60分通过）
- 不通过必须修改整卷后再进入下一卷

#### 第四级：全书终检（W22）
- 全书完本后进行
- 详见W22节点

### 【W23】全书终检

**执行步骤：**

**步骤 22.1：九项检查清单（逐项逐章检查）**

**① 情节逻辑检查**
- 逐章检查情节是否因果连贯
- 检查是否有"为了推进而推进"的强行情节
- 检查角色决策是否有合理动机
- 检查是否有"机械降神"（突然出现不合理的力量/信息解决问题）

**② 角色行为一致性检查**
- 逐角色检查行为是否符合性格标签
- 检查角色成长是否有铺垫
- 检查配角是否有"工具人化"倾向
- 检查对话是否符合角色身份

**③ 力量体系自洽检查**
- 逐章检查能力使用是否符合设定
- 检查是否有"忘记限制条件"的情况
- 检查境界/等级是否合理
- 检查战力是否崩坏

**④ 地图/势力一致性检查**
- 检查地点描述是否前后一致
- 检查势力立场是否合理变化
- 检查人物移动路线是否合理
- 检查时间地点是否冲突

**⑤ 时间线检查**
- 建立完整时间轴
- 检查日期/季节/年龄是否连贯
- 检查事件先后顺序是否合理
- 检查"过了三天"和实际描写是否矛盾

**⑥ 伏笔回收检查**
- 列出所有伏笔清单
- 标注每个伏笔的回收状态（已回收/未回收/计划回收）
- 检查伏笔与回收之间的逻辑关联
- 检查是否有"挖坑不填"

**⑦ 悬念解答检查**
- 列出所有悬念清单
- 标注解答状态
- 检查解答是否合理（不突兀）
- 检查是否有"悬念消失"（忘了回答）

**⑧ 文风统一检查**
- 检查不同章节文风是否一致
- 检查对话风格是否统一
- 检查叙述视角是否一致（除非有意切换）

**⑧.1 去AI味终检（调用 novel-de-ai）**
> 调用 Skill: novel-de-ai 对全文进行AI痕迹检测

- 扫描全文，检测24种AI味特征
- 按P0/P1/P2分级
- 重点检查：
  - [ ] 无"总结代替呈现"（情绪通过行为/环境呈现）
  - [ ] 无空洞形容词堆砌
  - [ ] 无排比堆砌（三段式法则）
  - [ ] 无万能连接词（然而/因此/综上所述）
  - [ ] 无被动语态泛滥
  - [ ] 无虚假主体/无生命拟人化
  - [ ] 无专家口吻/说教腔
  - [ ] 无道德评判/强行升华
  - [ ] 无过度解释
  - [ ] 对话能区分角色（遮住"他说/她说"能分清）
  - [ ] 对话有潜台词（不直接说内心想法）
  - [ ] 场景有五感描写
  - [ ] 情绪有身体反应支撑
  - [ ] 无机械重复（同词200字内≤3次）
  - [ ] 无翻译腔/长定语链
  - [ ] 无过度限定/软化语气
  - [ ] 无虚假乐观/通用积极结论
  - [ ] 无同义词循环（主人公/主角/英雄切换）
  - [ ] 无破折号/冒号滥用
  - [ ] 无小红书/自媒体腔
  - [ ] 无暴力执行力腔
- 评分：直接性+节奏+信任度+真实性+精炼度，总分≥40/50

**⑨ 文字校对**
- 错别字检查
- 语病检查
- 标点符号检查
- 段落格式检查

**步骤 22.2：整体阅读**
